# QCurl 3.0.0-rc.1 release notes draft

## 发布口径

`3.0.0-rc.1` 是 Core public surface reset 的 RC 候选，不是整个源码树的 Stable 宣告。

默认稳定候选范围只包含 Core install surface。Blocking Extras、Test Support、Other Extras 和 Preview 必须显式 opt-in，不随默认 Core 隐式承诺。

## Core 候选内容

- 默认 Core HTTP async manager / request / reply surface
- `QCCookieAsyncResult.h` cookie async result 值类型
- TLS / proxy / timeout / retry / cache policy 配置
- scheduler、Cache lookup、Multipart builder、body helper、transfer job
- logger、default logger、cancel token、middleware base
- connection pool config / manager

## 明确不进入默认 Stable 的内容

- Blocking Extras 同步 value-result client
- Test Support mock / capture helper
- Diagnostics schema、外部命令和权限模型
- WebSocket 连接阶段语义、公开字段配置类型和性能/压力合同
- 未绑定 benchmark 版本、依赖版本、网络环境和日期的性能数字

## 发布前阻塞项

Stable shared library 仍需完成并归档：

- ABI baseline 与 ABI diff gate
- full release gate
- libcurl capability matrix
- WebSocket / Diagnostics 独立 Stable 提升 checklist
- 大职责文件的拆分或 Stable 前 blocker 记录
