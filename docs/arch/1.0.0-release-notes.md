# QCurl 1.0.0 release notes

本文是最终 GitHub Release 的候选说明草稿；在 tag 与 GitHub Release 实际创建前，不表示远程 release 已完成。

`QCurl 1.0.0` 是项目首个 stable line。它稳定默认 Core install surface，同时保留 Blocking Extras、Test Support 和 Other Extras 的显式 opt-in 边界。

## Stable scope

Core / Stable 覆盖默认 `find_package(QCurl)` / `QCurl::QCurl` consumer contract：

- 异步 HTTP Core API：`QCNetworkAccessManager`、`QCNetworkRequest`、`QCNetworkReply`。
- 请求配置：TLS、proxy、timeout、redirect、transfer、retry、cache policy。
- HTTP method / version / error / priority types。
- lane-aware scheduler。
- Cache lookup、Multipart/body helper、transfer jobs。
- logger、default logger、cancel token、middleware base。
- connection pool config / manager。
- Core cookie API 使用 `QCurl::QCCookie`、`QCCookieOperationResult` 与
  `QCCookieExportResult`；默认 Core consumer 不需要 QtNetwork cookie 类型或
  `Qt6Network` 链接依赖。

## Package-shipped non-default surfaces

- Blocking Extras：同步 value-result client、受限内存 body、download-to-device 和 blocking cookie store。
- Test Support / Explicit opt-in：mock/capture/test-support helper，仅测试 opt-in。
- Other Extras / Preview：Diagnostics、Middleware Extras、WebSocket；它们可随包发布，但不属于 1.0.0 Core Stable 承诺。

## Compatibility policy

1.0.0 是新的 stable identity。旧历史版本和 RC 叙事保留为内部参考，不进入当前 release history 主线；本发布不提供旧 `2.x` / `3.x` 兼容层。

Core cookie public API 是 1.0.0 hard-break surface 的一部分：旧 QtNetwork cookie
类型不会作为兼容 overload、alias、shim 或 wrapper 保留。下游应迁移到
`QCurl::QCCookie` 并按当前 `QCurl::QCurl` / `QCurl::OtherExtras` target 边界重新构建。
