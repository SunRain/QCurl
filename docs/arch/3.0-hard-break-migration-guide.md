# QCurl 3.0 hard-break migration guide

> 本文记录 `3.0.0-rc.1` 的默认 Core scope、破坏性边界和下游迁移入口。它不把 Blocking Extras、Test Support、Other Extras 或 Preview 写成默认 Stable 承诺。

## 默认 Core scope

默认 `find_package(QCurl CONFIG REQUIRED)` / `QCurl::QCurl` consumer 只承诺 Core install surface。Core 以 `src/CMakeLists.txt` 的 `QCURL_INSTALL_HEADERS` 加生成头 `QCurlConfig.h` 为准，当前包括：

- `QCNetworkAccessManager` 与异步 `send*()` 路径
- `QCCookieAsyncResult` 中的 `QCCookieOperationResult` / `QCCookieExportResult`
- `QCNetworkRequest`、`QCNetworkReply` 和 HTTP method / version / error / priority types
- TLS / proxy / timeout / retry / cache policy 配置
- lane-aware scheduler
- Cache lookup API
- Multipart builder 与 body helper
- transfer/download/resumable job types
- logger、default logger、cancel token、middleware base
- connection pool config / manager

## 破坏性迁移点

QCurl 3.0 允许 API / ABI hard-break，不提供旧 API alias、wrapper 或过渡层。

迁移要求：

- 使用 `QCNetworkAccessManager::send*()` 异步路径作为 Core 入口。
- 同步 value-result client 改为显式 Blocking Extras，不能依赖默认 Core 安装。
- Mock / capture / test helper 改为显式 Test Support，不能用于生产 Core contract。
- Diagnostics 与 WebSocket 改为 Other Extras / Preview，不能被默认 Core consumer 隐式 include。
- Cookie async 结果使用 `QCCookieOperationResult` / `QCCookieExportResult` accessor API，不依赖 public fields。

## 非默认 surface

| surface | 当前发布身份 | 迁移要求 |
| --- | --- | --- |
| Blocking Extras | 显式 opt-in | 仅同步 value-result 工具使用，不能混入 Core 快速开始 |
| Test Support | 显式 opt-in | 仅测试程序安装和链接，不作为生产运行时能力 |
| Diagnostics | Other Extras | `ping/traceroute` 权限、外部命令和 details schema 不属于默认 Core Stable |
| WebSocket | Other Extras / Preview | 需要独立 install surface、consumer smoke、ABI 检查、异步连接语义和压力/延迟证据后才可提升 Stable |

## ABI / release gate

Stable shared library 发布前必须同时具备：

- `scripts/qcurl_abi_gate.py` 生成的 ABI baseline
- `scripts/qcurl_abi_gate.py diff` 的 ABI diff 证据
- `scripts/run_release_gate.py --tier full` 的 full release gate 证据
- `build/libcurl_consistency/reports/capabilities.json` 的 capability matrix

只通过 public-api / public-api-slow 不等于 Stable ABI ready。
