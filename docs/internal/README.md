# Internal documentation

This directory keeps maintainer-only history, status boards and archived release material that should not appear in the first-public-release documentation flow.

## Entries

- `pre-1.0-history.md` — compact index for old date-based changelog and pre-1.0 development history.
- `archived-release/` — raw pre-1.0 history plus old RC, 3.0, hard-break review and task-log documents kept for audit/reference only.
- `maintainer-backlog/` — internal maintainer status boards that are not public testing or release entrypoints.

Current public release documents live under `README.md`, `CHANGELOG.md`, `docs/README.md`, `docs/user/`, and the current `docs/arch/1.0-*` contract files.
