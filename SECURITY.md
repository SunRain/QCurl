# 安全策略（SECURITY）

## 报告安全漏洞

请**不要**在公开 Issue 中直接披露安全漏洞细节。

优先方式（如仓库已启用）：

1. 使用 GitHub Security Advisories（私密披露）

备选方式：

2. 发送邮件至：`qcurl@example.com`

请在报告中提供：

- 影响版本/提交（如可用）
- 复现步骤与 PoC（如可公开的最小化示例）
- 风险评估（可选）

## 支持版本范围（声明）

我们通常优先修复：

- 主分支（main/master）
- 最近一个稳定发布版本（如存在）

> 如你依赖更老版本，请在报告中说明升级约束，我们会尽力给出缓解建议。
